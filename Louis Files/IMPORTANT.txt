HacKing Gemmini (Interesting parameters):
    •	"memory-access" and "execute" instructions happen concurrently, in different regions of the hardware.
    •	Systolic array dimensions (tileRows, tileColumns, meshRows, meshColumns): The systolic array is composed of a 2-level hierarchy, in which each tile is fully combinational, while a mesh of tiles has pipeline registers between each tile.
    •	Dataflow parameters (dataflow): Determine whether the systolic array in Gemmini is output-stationary or weight-stationary, or whether it supports both dataflows so that programmers may choose between them at runtime.
    •	Scratchpad and accumulator memory parameters (sp_banks, sp_capacity, acc_capacity): Determine the properties of the Gemmini scratchpad memory: overall capacity of the scratchpad or accumulators (in KiB), and the number of banks the scratchpad is divided into.

Where to change parameters:
    •	chipyard/generators/gemmini/src/main/scala/gemmini/CustomConfigs.scala

    •	The examples in CustomConfigs.scala show how to change parameters.

After Changing parameters (Do this first):
1)	cd ~/SsysArch/gemmini/chipyard
2)	source env.sh
3)	export JAVA_TOOL_OPTIONS="-Xms2g -Xmx16g -XX:+UseG1GC" 
        -> -Xms2g: Initial Java heap size = 2 GB.
        -> -Xmx16g lets Java to use 16 Gig RAM (Maximum Java heap size = 16 GB). If 16 GB is not enough for compiling, increase.
        -> -XX:+UseG1GC -> Use the G1 garbage collector.
4)	cd ~/SsysArch/gemmini/chipyard/sims/verilator
5)	make clean
6)	make CONFIG=NoDebugCustomGemminiSoCConfig -j1

Run simulation:
1) cd ~/SsysArch/gemmini/chipyard/generators/gemmini/software/gemmini-rocc-tests
2) ./build.sh
3) cd ~/SsysArch/gemmini/chipyard/sims/verilator
5) ulimit -s unlimited
4) ./simulator-chipyard.harness-NoDebugCustomGemminiSoCConfig \
 ../../generators/gemmini/software/gemmini-rocc-tests/build/bareMetalC/my_test-baremetal \
  +max_core_cycles=2000000000
